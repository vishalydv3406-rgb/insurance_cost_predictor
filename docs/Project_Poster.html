Insurance AI Pro+

End-to-End Data Science Project Lifecycle

Advanced Premium & Claim Prediction System using Machine Learning

Problem Definition

Objective: Build an intelligent, multi-domain insurance quotation system to accurately predict insurance premiums or claim amounts based on demographic, physical, and historical user attributes.

  • Regression: Predicting continuous Premium/Claim amounts.
  • Classification: Categorizing candidate profiles as "High" vs "Low Risk".
  • Explainability (XAI): Identifying core factors to help users reduce costs.

Data Collection & Processing

Gathered via statistical distributions specifically mirroring real-world epidemiological and physical correlations (e.g., BMI follows a Normal Distribution; Children follow a Poisson Distribution).

  • Scope: 8 distinct datasets (Health, Life, Motor, Business, Property, Travel, Specialty, Claims).
  • Scale: Over 120,000 generated records total.
  • Handling: Median imputation for numericals, constant for categoricals. Outliers mathematically clipped.

Feature Engineering

  • Encoding: LabelEncoder (binary), OneHotEncoder (multi-class).
  • Scaling: StandardScaler applied to normalize variance.
  • Interaction: PolynomialFeatures (degree=2) to capture synergistic risk factors (e.g., High BMI + Smoking).
  • Dynamic Factor: Engineered a No Claim Bonus (NCB) compounding discount feature for fidelity.

Algorithms Selected

We evaluated multiple supervised learning algorithms crossing varying degrees of complexity to find the optimal global fit:

  • Linear/Distance: Linear Regression, SVR, KNN.
  • Tree-based Ensembles: Decision Trees, Random Forest.
  • Boosting: Gradient Boosting, XGBoost.

Since premium rules are non-linear and conditional (e.g., a smoker penalty is a multiplier), Gradient Boosting Engines were expected to dominate.

Application & Deployment

The core pipeline was serialized via `joblib` (.pkl files) and integrated into a real-time responsive web UI via Streamlit.

  • Interactive Prediction: Real-time premium costing.
  • XAI Savings Advisor: Providing users counter-factual adjustments (e.g. "If you stopped smoking, you save 20%").
  • Database Logging: Queries logged to NoSQL MongoDB instance to track data drift over time.

Results & Discussion

The hypothesis that Ensemble Boosting mechanisms capture insurance logic best was definitively proven. Linear models suffered dropping 5-10% of variance mapping, unable to capture intersecting compound risks efficiently.

Classification thresholding managed a 99.71% Accuracy sorting distinct risk profiles.

Conclusion & Future Enhancements

Conclusion: We built a highly viable Data Science pipeline mapping multi-domain insurance pricing effectively.
